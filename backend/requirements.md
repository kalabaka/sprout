## 1. 引言

### 1.1 目的

本文档旨在全面描述面向大学生的个人学习规划智能体系统的功能需求、非功能需求及技术规格，为系统开发提供完整的需求依据。该系统旨在帮助大学生科学规划学习任务、合理分配学习时间、持续提高学习效率，解决现有学习方法缺乏系统性学习路径规划和动态反馈机制的问题。

### 2.1 范围

本系统是一个基于前后端分离架构的Web应用系统，服务对象为普通高等学校在校大学生。系统通过智能体技术为学生提供个性化的学习规划、执行、评估和动机干预全流程支持。

### 2.2 定义

| 术语 | 定义 |
|------|------|
| 智能体(Agent) | 能够感知环境并采取行动以实现目标的软件实体 |
| 学习路径 | 基于知识依赖关系生成的有序学习内容序列 |
| 学习计划 | 对学习目标、任务、时间等要素的统筹安排 |
| 绩效评估 | 对学习效果的量化分析与评价 |
| ARCS模型 | Attention（注意）、Relevance（相关）、Confidence（信心）、Satisfaction（满足）动机模型 |

## 2. 系统概述

### 2.1 目标和愿景

**核心目标**：为大学生提供科学、高效、个性化的学习规划支持系统，帮助学生实现学习目标的最大化达成。

**关键价值**：
- 实现学习计划的可规划、可执行、可评估
- 提供动态的学习路径推荐和调整机制
- 通过动机干预提升学习坚持度
- 数据驱动的学习效果分析

### 2.2 系统用户

| 用户角色 | 描述 | 权限范围 |
|--------|------|----------|
| 学生 | 系统主要使用者 | 创建学习计划、执行学习任务、查看进度和分析报告 |
| 管理员 | 系统维护者 | 用户管理、系统配置、数据查看 |

### 2.4 系统架构

**技术架构**：前后端分离
- **前端**：Vue.js / React
- **后端**：Node.js + Express
- **数据库**：MySQL

**智能体架构**：双Agent协作
- **接口Agent**：负责与用户交互，收集输入信息
- **核心规划评估Agent**：负责学习路径生成、计划制定、绩效评估、动机干预

### 3.5 数据分析模块

#### 3.5.1 绩效评估模型

| 需求ID | F-DA-001 |
|--------|----------|
| 需求描述 | 多维度学习效果评估 |
| 评估维度 | 时间偏差、任务完成率、准确率 |
| 计算公式 | 综合评分 = w1×(时间偏差得分) + w2×(任务完成得分) + w3×(准确率得分) |

**评估指标**：
- **时间偏差**：实际用时与预估用时的偏差程度
- **任务完成率**：已完成任务占总任务的百分比
- **准确率**：学习内容掌握程度评估

#### 3.5.2 学习报告生成

| 需求ID | F-DA-002 |
|--------|----------|
| 需求描述 | 生成阶段性学习分析报告 |
| 报告内容 | 学习时长、任务完成情况、绩效评分、改进建议 |
| 输出格式 | 文本/图表 |

#### 3.5.3 数据统计

| 需求ID | F-DA-003 |
|--------|----------|
| 需求描述 | 统计学习数据 |

```
                                                        │
                    ┌────────────┐     ┌────────────┐     ▼
                    │  调整学习 │ ──▶ │  重新生成 │ ── ─ ─ ─ ─ ─
                    │   计划   │     │   计划   │
                    └────────────┘     └────────────┘
```

### 7.2 学习执行流程

```
1. 任务获取
   ↓
2. 开始任务 → 记录开始时间
   ↓
3. 执行学习
   ↓
4. 完成任务 → 提交完成信息
   ↓
5. 绩效评估 → 计算时间偏差/完成率/准确率
   ↓
6. 生成评估报告
   ↓
7. [是否异常?] → 是 → 触发动机干预
              ↓ 否
          继续/完成
```
